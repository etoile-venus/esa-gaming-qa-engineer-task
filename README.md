# ESA Gaming QA Engineer Task

This repository contains my work for the ESA Gaming QA Engineer test task.

The goal of this repository is to keep the QA work clear, reviewable, and easy to reproduce. It includes manual testing documentation, image difference analysis, and placeholders for optional Cypress UI automation and Postman/Newman API testing.

## Current Status

The repository is still in progress.

Completed so far:

- Repository structure for manual testing, Cypress automation, and Postman API testing.
- Image differences analysis for the visual comparison task.

Planned next:

- Application behavior analysis.
- Manual test cases.
- Bug report documentation.
- Optional Cypress automation for selected stable UI scenarios.
- Optional Postman collection and Newman run instructions.

## Application Under Test

The manual and automated QA work is focused on:

https://practicesoftwaretesting.com/

## Repository Structure

- `manual-testing/image-differences/` - image comparison notes, reference assets, and final analysis.
- `manual-testing/test-cases/` - manual test cases.
- `manual-testing/bug-reports/` - bug reports.
- `cypress-automation/` - optional Cypress UI automation.
- `postman-api/collection/` - optional Postman collections.
- `postman-api/environment/` - optional Postman environments.
- `postman-api/reports/` - optional Newman reports.

Each main folder has its own `README.md` file with a short explanation.

## Documentation Style

Public documentation is written in simple and clear English. The intention is to make the work easy to understand for reviewers without adding unnecessary complexity.
